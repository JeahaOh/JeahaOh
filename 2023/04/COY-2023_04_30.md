대용량 데이터 처리 기능 고도화

- 2022.08.15 ~ 2022.08.31
- 하루 평균 2,700 건의 배송을 추적하는 기능의 고도화.
- Spring Async Annotation과 MyBatis Execute type BATCH 기능으로 처리하여 소요시간 약 60% 감소.
- 물류센터 및 각 브랜드 별 매장에서 발송하는 물류가 여러 택배사를 이용. 택배사별로 배송 상태 조회 API의 응답 값의 구조가 다르기 때문에 스윗트래커 API를 이용하여 배송상태를 조회. 배송을 건별 조회하기 때문에 약 30분 걸리던 배송 추적 프로세스를 멀티프로세스 비동기 조회 및 여러 쿼리 문을 묶어서 Transaction을 관리하도록 프로세스를 변경하여 소요시간을 약 17분으로 감소.
- Transactional Annotation의 REQUIRED_NEW 속성 사용시 예외처리가 잘 되어있지 않으면 부모 트랜잭션까지 Rollback 되는 문제가 발생, 부모/자식 프로세스 간 트랜잭션을 독립적으로 처리 되도록 예외처리.
- 절차지향적으로 개발된 기존의 물류/배송 프로세스를 참고하여 처리대상 조회, 전처리, API 통신, 응답 값 정재, 데이터 저장의 프로세스 등을 객체지향 구조로 개발.

## 지원하신 직무를 잘 수행할 수 있다고 생각하는 이유를, 보유한 전문지식 및 스킬과 관련하여 기술하시오. (max 2,000)

커머스 전문지식과 스킬을 보유하고 있다고 자신합니다. 약 4년간의 개발 경력과 그중 1년 8개월간의 아이디룩 자사 온라인 쇼핑몰의 운영 및 고도화 경험을 가졌습니다. 이를 자세히 설명하면 다음과 같습니다.

1. 커머스 도메인에 대한 이해
  커머스 도메인에는 상품/전시, 마케팅, 주문/결제/CS, 배송, 회원 관리 등 세분된 도메인이 있고 이에 따른 다양한 기능이 필요합니다. 이러한 기능을 개발하고 운영하기 위해서는 커머스 도메인에 대한 이해도가 높아야 합니다.  
  저는 마케팅, 상품/전시, 배송 도메인에 대한 서비스 고도화 및 운영 경험을 바탕으로 커머스 개발자로서 필요한 도메인 지식을 쌓아 왔습니다.

2. 자바와 스프링에 대한 높은 이해도
  저는 자바와 스프링 프레임워크를 이용한 웹 애플리케이션 개발 및 운영 그리고 고도화를 중심으로 커리어를 만들어 왔습니다. 신규 프로젝트 개발과 레거시 프로젝트의 유지보수 그리고 기능 고도화의 경험으로 현재 직면한 문제 파악과 적절한 기술을 적용해 문제를 해결하는 다양한 경험을 했습니다.
  제가 경험한 다양한 프로젝트는 기술 부채가 많았습니다. 절차 지향적이거나 중복된 프로세스로 작성된 소스가 많았습니다. 유지보수, 코드의 재활용성, 확장에 대한 유연성이 낮았습니다.
  저는 틈틈이 프로젝트의 기술 부채를 해결하기 위해 클린 코드의 철학에 입각한 리팩토링을 시행합니다. 스프링의 DI를 활용하여 중복 코드를 가진 메소드를 별도의 클래스로 분리하였습니다. 이 클래스는 인터페이스를 구현하도록 설계하고, 필요한 서비스 클래스에서는 이 인터페이스의 구현체를 주입받아 사용하도록 하였습니다. 이를 통해 중복된 코드를 가진 클래스들은 해당 인터페이스의 구현체를 주입받아 공통으로 사용할 수 있는 코드를 별도의 클래스로 분리했습니다. 또한 AOP를 활용하여 로깅 기능을 분리했습니다. 타 서비스와 API 통신할 때마다 요청과 응답 값을 별도로 로깅 할 수 있도록 어드바이스를 작성하고 이를 서비스 클래스에 적용해 비즈니스 로직과 로깅 로직의 관심사별로 기능을 분리했습니다. 이러한 방식으로 DI와 AOP를 활용하여 코드의 중복을 줄여 기술 부채를 줄이고 유지보수성을 향상시켰습니다.

3. DB 관리 및 SQL 작성 능력
  커머스 시스템의 전시, 검색은 대량의 데이터를 다루기 때문에 데이터베이스에 대한 이해와 쿼리 작성 능력이 중요합니다. 저는 SQL을 이용한 RDBMS 설계 및 최적화 그리고 복잡한 쿼리를 작성할 수 있는 능력을 보유하고 있습니다. 현재 운영 중인 아이디룩 몰에서 최근 상품 목록 페이지의 쿼리 개선을 위해 인덱스 개선, 조회 조건 최적화, 서브쿼리 최적화, view와 mview 활용, 캐싱 활용 등의 다양한 최적화 작업을 오라클 DB의 GATHER_PLAN_STATISTICS 힌트와 DBMS_XPLAN에 저장된 실제 실행 계획을 비교 분석해 해당 쿼리의 실행 비용을 {4,095 CPU COST, 24,418 BUFFERS}에서 {1,047 CPU COST, 12,913 BUFFERS}으로 개선했습니다.

4. AWS 클라우드 서비스 이용과 그 외 인프라 고도화에 대한 경험
  작년 상반기에 현재 운영 중인 아이디룩 몰의 인프라를 IDC에서 AWS 클라우드 서비스로 전환하였습니다. OS 파일 시스템에서 AWS S3로, IDC CDN에서 AWS Cloud Front로, AWS Elastic Cache for Redis를 이용한 Session Clustering 구성 등의 인프라 개선 경험이 있습니다. 또한 다수의 프로젝트에서 Apache Tomcat, Apache Http Server, nginX 등의 미들웨어 인프라 구성과 어플리케이션 실행에 사용하는 JVM 튜닝 등의 경험이 있습니다.
