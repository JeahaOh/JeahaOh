## 랜덤 마일리지 지급?

가장 최근에 한거?

- 창립 35주년 기념 이벤트
- 마케팅 팀에서 회원에게 룰렛을 돌려 렌덤으로 당첨된 마일리지를 지급하는 기능을 원했음.
- 마케팅에서 요구한 조건은 1000원 50%, 2000원 30%, 3000원 15%, 5000원 5%의 가중치로 랜덤 지급하길 원했음.
- 디자인, 퍼블팀과의 빠른 협업을 위해 룰렛은 HTML canvas에 javascript로 생성하여 룰렛판을 클릭시 회전하도록 만들어 전달 하였음.
- BE에서 지급할 적립금을 추첨하여 응답, 화면에서 룰렛을 돌려 해당 금액이 나오도록 만들었음.
- 이벤트_적립금을 관리하기 위한 테이블 생성
  - 이벤트 id
  - 옵션 번호
  - 적립금 id
  - 지급 금액
- 이벤트_적립금_지급을 관리하기 위한 테이블 생성
  - 지급 번호
  - 적립금 id
  - 지급 받은 회원 id
  - 지급일
- 이벤트_적립금_지급 테이블에 비중에 맞게 지급번호, 적립금 ID를 10만 row를 생성해 밀어 넣었음.
- 가중치가 있는 랜덤 추첨 로직은 난수 / (가중치(남은 쿠폰수 / 총 쿠폰 수 * 100))를 Math.log 메소드로 뽑았음.
- 시간이 걸린거지 어려운건 아님...
