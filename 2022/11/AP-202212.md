## 직무기술서 (지원 직무와 관련하여 주요 수행업무 및 실적을 기준으로 상세히 기술해 주십시오) (3000 자)

```txt
  JD + 경력 + 성과
    핵심 경력 사항 최신순
    성과에 대한 객관적 자료
    뭐를 개발하였으며 어떻게 개선 되었다
    업무 강점 및 경험에서 얻은 전문가적 견해
    텍스트와 이미지
```

저는 현재 아이디룩의 이커머스 개발팀 주임으로 근무하며 자사 쇼핑몰(아이디룩 몰, 아페쎄몰) 백엔드 개발실무를 맞고 있습니다. 상세 업무는 쇼핑몰 웹 서비스의 오류수정, 기능개선, MD의 운영지원, 통계 데이터 추출, 장애 대응 및 해결, CS/배송 업무처리 기술 지원 그리고 인프라 관리를 하고 있습니다.

최근에는 자사 물류창고의 인력 수급의 문제로 사용자의 주문 완료 후 출고까지 3~5일이 걸리는 이슈를 개선하기 위해 3자 물류를 연계하여 1~2일이면 출고하여 배송이 시작되도록 프로그램을 비즈니스 로직을 변경하였습니다. 하지만 3자 물류사와 API 협의가 잘 이루어지지 않아 결품 처리, 합포장 이슈가 생겨 개선 중입니다.

10월 초에는 이커머스 MD 운영팀에서 이벤트를 진행하였고 이벤트의 결과로 1만여 명에게 마일리지를 지급하려 했으나 오류가 발생하였습니다. 원인을 찾아보니 기존의 마일리지 지급 쿼리는 IN 절에 대상 회원의 ID를 넣어 실행하는 형식이었고, Oracle DB 11G에서는 1천 건 이상의 값을 처리할 수 없어서 생기는 문제였습니다. 이 문제를 인지하고 다량의 데이터를 다루는 다른 기능들을 살펴보니 역시 IN 절에 식별자를 담아 실행하거나, 한 번에 한 건씩만 데이터의 삽입/수정 트랜잭션을 수행하도록 만들어져 있었습니다. 트랜잭션이 한번 실행되는 소요 시간에 대량의 데이터를 처리하도록 효율을 개선하고자 Java 언어에서 관계형 데이터베이스의 작업을 지원하는 MyBatis의 Execute Type BATCH 설정을 사용할 수 있도록 비즈니스 로직을 개선하였습니다. BATCH DML 또한 한번의 트랜잭션에 처리할 수 있는 데이터양이 제한되어 있음을 알게 되어 데이터의 목록에 페이징 처리를 하여 한 트랜젝션에 일정 수의 데이터를 처리하도록 기능개선을 하였고 마일리지/쿠폰 지급, 쿠폰 생성 등 대량의 데이터를 처리하는 기능의 소요 시간을 1/4로 줄이는 성과를 이뤘습니다.

9월에는 이벤트/프로모션 기능개선을 하였습니다. 기존 소스는 쿠폰 다운로드, 사다리 타기, 응모, 랜덤 추첨 등의 프로모션을 한 번에 한 개의 콘텐츠만 생성하여 사용할 수 있도록 개발되었습니다. 마케터나 운영 MD의 기획으로 이런 기능을 다중 콘텐츠로 진행하게 되면 적게는 3일~2주의 개발기간이 필요했습니다. 이에 따라 개발, 마케팅, 운영 MD 세 개의 팀에서 소비되는 공수가 많다고 판단하였고 이벤트 기능의 통합을 제안하였습니다. 주도적으로 2주간의 작업을 진행하였고, 다중 콘텐츠 프로모션의 진행 전후로 소비되는 공수가 없도록 기능을 개선했습니다.

2022년 상반기에는 IDC에서 AWS로 인프라 전환을 했습니다.
해당 프로젝트에서 저는 Redis를 사용하여 어플리케이션의 세션 클러스터링 구축, OS File System 및 Static Resources의 S3 Cloud Front로 전환 등을 맡아 작업하였습니다.
어플리케이션을 IDC에서 운영하던 당시에는 2대의 서버가 세션 클러스터링 기능 없이 작동했습니다. 해당 문제로 사용자는 결제 시 로그인이 풀리는 등의 말이 안 되는 UX를 겪었고, 이와 관련된 CS 인입이 많았습니다. 클라우드 환경으로 전환하면서 Redis에 세션 클러스터링을 구축하였고 세션과 관련된 CS는 그 후 사라졌습니다.
또한 상품 후기로 멀티파트 업로드하거나 상품 이미지를 등록할 때 OS File System을 이용하여 파일을 저장하던 비즈니스 로직을 AWS S3 SDK를 이용하도록 전환하였고, static resources 들을 웹서버에서 직접 호스팅하는 시스템 구조를 Jenkins에서 빌드 시 S3에 FTP로 저장하도록 설정, Cloud Front에서 호스팅하도록 개선하였습니다. 이 개선 작업으로 6,000원/1Mbps 종량제로 IDC에서 운영하던 CDN 서비스를 3,600원/1Mbps의 AWS Cloud Front 서비스로 전환하여 월평균 160만 원 정도의 비용을 절감하였습니다.

그 외에 월 마감 정산을 위한 데이터 추출의 자동화, 마케팅 LMS 발송을 위한 고객 데이터 추출 기능 개발, 파편화된 소스/중첩 분기문등의 개선으로 유지보수성을 향상하고 기술 부채를 줄이는 데 기여했습니다.
